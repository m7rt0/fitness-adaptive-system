README - Интелигентна адаптивна система за фитнес тренировки

1. Използвани технологии

Приложението е разработено чрез Node.js, Express, MySQL, HTML, CSS и JavaScript.

2. Структура на проекта

Проектът е организиран в няколко основни директории:

backend - съдържа сървърната логика на приложението.

frontend - съдържа потребителския интерфейс.

database - съдържа SQL файловете за създаване и управление на базата данни.

3. Подготовка на базата данни

За работа на приложението е необходима MySQL база данни.

В MySQL Workbench се изпълнява файлът:

database/schema.sql

След това се изпълнява файлът:

database/seed.sql

Файлът schema.sql създава структурата на базата данни, а seed.sql добавя начални данни, необходими за работата на системата.

4. Конфигурационен файл

В папката backend трябва да се създаде .env файл по примера от .env.example.

Примерна структура:

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=fitness_adaptive_system
PORT=3000

В DB_PASSWORD се въвежда реалната MySQL парола на локалната машина.

5. Стартиране на backend сървъра

Отваря се Command Prompt или терминал в папката:

backend

При първо стартиране на проекта се изпълнява командата:

npm install

Тази команда инсталира необходимите зависимости.

След това backend сървърът се стартира с командата:

npm.cmd start

След успешно стартиране сървърът работи на адрес:

http://localhost:3000

6. Стартиране на frontend частта

HTML файловете от папката frontend се отварят в браузър.

Началният екран на приложението е страницата за вход и регистрация.

След успешен вход потребителят може да използва основните функционалности на системата:

генериране на тренировъчна програма;

стартиране на тренировка;

записване на изпълнени серии;

въвеждане на обратна връзка;

получаване на адаптивна препоръка;

преглед на историята на тренировките.

7. Помощни SQL файлове

В папката database са включени и помощни SQL файлове:

delete_user.sql - изтрива конкретен потребител и свързаните с него данни.

reset_user_data.sql - изчиства тренировъчните данни на избран потребител.

test_user.sql - справка на потребители

Тези файлове са предназначени за тестване, поддръжка и демонстрация на приложението.

8. Основен работен процес

Потребителят създава профил чрез регистрация.

След това влиза в системата чрез потребителско име или имейл и парола.

След успешен вход може да генерира тренировъчна програма чрез избор на цел, ниво и брой тренировъчни дни.

След създаване на програма потребителят може да стартира тренировка, да избере тренировъчен ден и да запише реално изпълнени серии.

След приключване на тренировката се въвежда обратна връзка, включваща оценка на умората и бележки.

На базата на въведените тренировъчни данни системата формира адаптивна препоръка.

Всички записани тренировки и препоръки могат да бъдат прегледани в страницата с историята.
